    Dépannage

    Cette documentation, qui retraduit le site 3DS Hacks Guide (alias Nintendo Homebrew), fournit des conseils de dépannage pour les problèmes courants.

    Si les solutions proposées ne résolvent pas votre problème, nous vous invitons à rejoindre un des serveurs conseillés et de décrire votre problème
    ainsi que les étapes auquel vous avez déjà tentées.

    Installation de boot9strap (MSET9)

    Python 3 n'est pas installé

    Python n'est pas installé sur l'ordinateur que vous utilisez.
    Téléchargez-le à partir du site web de Python, double-cliquez sur
    le programme d'installation et suivez les instructions pour installer
    Python. Une fois Python installé, réessayez.

    HOME Menu extdata : Manquant !

    Veuillez allumer votre console avec votre carte SD insérée, puis vérifiez à nouveau l'état de MSET9.

    Si cela ne fonctionne pas, votre carte SD doit être formatée :

    1. Copiez tout ce qui se trouve sur la carte SD sur votre PC
    2. Formatez la carte SD (Windows, Linux, macOS))
    3. Copiez tout à nouveau
    4. Recommencez à partir de la partie 1 Étape 7
    Éditeur Mii extdata : Manquant !

    Les données Mii Maker n’ont pas été trouvées sur la carte SD.
    Veuillez allumer votre console avec votre SD insérée, puis lancer
    Mii Maker, puis vérifier à nouveau l’état du MSET9.

    Si cela ne fonctionne pas, votre carte SD doit être formatée :

    1. Copiez tout ce qui se trouve sur la carte SD sur votre PC
    2. Formatez la carte SD (Windows, Linux, macOS))
    3. Copiez tout à nouveau
    4. Recommencez à partir de la partie 1 Étape 8
    Base de données de titres : Non initialisée !

    Assurez-vous d’avoir réinitialisé la base de données des titres. + Veuillez allumer votre console avec votre SD insérée + Lancez les paramètres système et accédez à Data Management -> Nintendo 3DS -> Software -> Reset (image) + Cela n’effacera aucune de vos données + Si vous obtenez une invite de réinitialisation, après la réinitialisation, éteignez votre console et recommencez à partir de la partie 1 Étape 14

    Si vous ne recevez pas d’invite de réinitialisation, votre carte SD doit être formatée :

    1. Copiez tout ce qui se trouve sur la carte SD sur votre PC
    2. Formatez la carte SD (Windows, Linux, macOS))
    3. Copiez tout à nouveau

    4. Exécutez le script MSET9 :

      • Windows : double-cliquez MSET9-Windows.bat
      • macOS : double-cliquez sur MSET9-macOS.command et entrez votre mot de passe si vous y êtes invité
      • Linux : ouvrez une fenêtre de terminal, cd à la racine de votre carte SD, puis tapez python3 mset9.py et appuyez sur Entrée
    5. Saisissez le numéro correspondant au modèle et à la version de votre console, puis appuyez sur Entrée

    6. Tapez 2 puis appuyez sur Entrée pour vérifier l’état du MSET9

      • Cela créera à nouveau les bases de données factices
    7. Fermer la fenêtre de script MSET9
    8. Recommencez à partir de la partie 1 Étape 12.
    Erreur 01 : Impossible de trouver le dossier Nintendo 3DS

    Vous n’exécutez pas MSET9 à partir de la racine de la carte SD, ou la carte SD n’a pas le dossier Nintendo 3DS.

    N’oubliez pas que votre carte SD doit ressembler à ceci :

    Si la disposition de votre carte SD est correcte, il est fort probable qu’elle ne soit pas lue par votre console et qu’elle doive être formatée :

    1. Copiez tout ce qui se trouve sur la carte SD sur votre PC
    2. Formatez la carte SD (Windows, Linux, macOS))
    3. Copiez tout à nouveau
    4. Recommencer à partir du début de la partie 1
    Erreur 02 : Votre carte SD est protégée en écriture

    La protection en écriture est activée sur cette carte SD.
    Si vous utilisez une carte SD de taille normale, assurez-vous
    que le verrou est basculé en position verticale. Sinon, essayez
    d’éjecter et de réinsérer votre carte SD.

    Erreur 04 : Vous n’avez pas 1 ID0, vous avez (#) !

    Vous disposez de plusieurs dossiers ID0. Pour déterminer le bon dossier, suivez ces instructions :

    1. Renommez le dossier Nintendo 3DS en BACKUP_Nintendo 3DS
    2. Réinsérez votre carte SD dans votre console
    3. Allumez votre console

    4. Attendez que la console génère les données de la carte SD

      • Vos applications auront disparu. Ceci est normal et sera résolu plus tard
    5. Éteignez votre console
    6. Insérez votre carte SD dans votre ordinateur
    7. Naviguez vers le dossier Nintendo 3DS de votre carte SD

    8. Notez les premiers caractères du dossier que vous voyez

      • Il s’agit de votre véritable ID0, que nous conserverons dans le vrai dossier Nintendo 3DS
    9. Supprimez l’ID0 du dossier actuel de la Nintendo 3DS
    10. Déplacer le dossier true ID0 du dossier BACKUP_Nintendo 3DS vers le dossier Nintendo 3DS
    11. S’il existe, déplacez le dossier Private du dossier BACKUP_Nintendo 3DS vers le dossier Nintendo 3DS

    Une fois que vous avez fait cela, passez à l’étape 3 de la partie 1.

    Erreur 05 : Vous n’avez pas 1 ID1, vous avez (#) !

    Cela se produit lorsque vous utilisez plusieurs cartes SD avec la même 3DS, puis fusionnez le contenu de la carte SD. Les deux sont « valides », mais la 3DS ne peut en lire qu’un à la fois.

    Vous devrez déterminer lequel contient vos données. Tout d’abord, sauvegardez tous les dossiers, en cas de problème :

    Maintenant, supprimez tous les dossiers sauf un de votre carte SD. En règle générale, vous voudrez supprimer le(s) plus petit(s) site(s), car celui avec toutes vos données aura probablement plus d’espace occupé. Vous pouvez comparer quel dossier occupe le plus d’espace en faisant un clic droit dessus et en cliquant sur « Propriétés » :

    Votre carte SD devrait maintenant ressembler à ceci :

    Maintenant, insérez votre carte SD dans votre 3DS et allumez-la.
    Si vos données s’y trouvent toujours, vous avez supprimé le bon dossier
    et vous avez maintenant déterminé votre ID1 correct. Si tout a disparu,
    retournez dans le dossier de sauvegarde que vous avez créé précédemment
    et essayez l’autre dossier.

    Erreur 06 : Vous avez besoin d’au moins 16 Mo de libre

    Votre carte SD n’a pas assez d’espace pour déclencher MSET9.
    Libérez de l’espace et réessayez.

    À la fin de ce guide, vous aurez besoin d’au moins 1,3 Go pour effectuer une sauvegarde NAND,
    il est donc préférable de libérer au moins autant.

    Erreur 07 : Un ou plusieurs fichiers sont manquants ou mal formés !

    Un ou plusieurs fichiers dont MSET9 a besoin pour s’exécuter sont manquants ou corrompus.
    Téléchargez à nouveau la version MSET9 .zip et extrayez-la à la racine de votre carte SD,
    en remplaçant tous les fichiers existants, puis réessayez.

    Écran rouge après la réinsertion de la carte SD (Section II Étape 11)

    Il se peut qu’il vous manque SafeB9S.bin à la racine de votre carte SD ou que
    le fichier soit corrompu. Copiez-le à partir du .zip MSET9, en remplaçant tous
    les fichiers existants, puis suivez ces instructions pour supprimer le
    fichier déclencheur :

    1. Forcez l’arrêt de votre console en maintenant le bouton d’alimentation enfoncé pendant 20 secondes
    2. Insérez votre carte SD dans votre ordinateur

    3. Exécutez le script MSET9 :

      • Windows : double-cliquez MSET9-Windows.bat
      • macOS : double-cliquez sur MSET9-macOS.command et entrez votre mot de passe si vous y êtes invité
      • Linux : ouvrez une fenêtre de terminal, cd à la racine de votre carte SD, puis tapez python3 mset9.py et appuyez sur Entrée

    4. Saisissez le numéro correspondant au modèle et à la version de votre console, puis appuyez sur Entrée

      • L’état actuel doit afficher Injected
      • Si vous avez déjà supprimé le fichier déclencheur
        (ou si vous ne l’avez jamais injecté en premier lieu),
        l’état actuel affichera Prêt, et vous pourrez réessayer la Partie 2
    5. Tapez 4, puis appuyez sur Entrée
    6. Une fois que la fenêtre indique « Fichier de déclencheur supprimé », tapez 0, puis appuyez sur Entrée
    7. Réinsérez la carte SD dans votre console
    8. Allumez votre console
    9. Retour à la partie 2 Étape 1

    Alternativement, votre carte SD peut être mal formatée ou
    partitionnée. Après avoir supprimé le fichier déclencheur,
    formatez-le :

    1. Copiez tout ce qui se trouve sur la carte SD sur votre PC
    2. Formatez la carte SD (Windows, Linux, macOS))
    3. Copiez tout à nouveau
    4. Recommencer à partir du début de la partie 1

    Une exception s’est produite après le déclenchement de MSET9

    Vous devriez vérifier la présence de CFW.

    Voyants d’alimentation/de notification

    Ma console s’éteint lorsque j’essaie de l’allumer et/ou
    la LED de notification affiche une couleur au démarrage

    Il y a un problème avec votre fichier boot.firm. Si vous
    utilisez boot9strap 1.4, la LED de notification de votre
    3DS peut clignoter d’une certaine couleur. Cette couleur
    est utilisée pour diagnostiquer les problèmes impliquant
    votre fichier boot.firm sur la carte SD ou la mémoire interne.
    Sur les anciennes versions de boot9strap, le voyant bleu s’éteint
    presque immédiatement lorsque vous essayez d’allumer la console.

    Si le voyant de notification clignote :

    • Blanc : Votre 3DS n’a pas pu trouver boot.firm sur votre carte SD ou sur la mémoire interne.
    • Magenta : Votre 3DS n’a pas réussi à trouver boot.firm sur votre carte SD. Il a pu trouver boot.firm sur la mémoire interne, mais le fichier est corrompu.
    • Rouge : Votre 3DS a pu trouver boot.firm à la fois sur votre carte SD et sur la mémoire interne, mais les deux fichiers sont corrompus.

    Vous pouvez obtenir un nouveau fichier boot.firm en téléchargeant
    la dernière version de Luma3DS, en l’extrayant et en plaçant boot.firm
    à la racine de votre carte SD. Si votre fichier boot.firm est constamment
    détecté comme corrompu, vous pouvez vérifier que votre carte SD ne contient
    pas d’erreurs (Windows, Linux ou macOS). Notez également que le 3DS a tendance
    à avoir des problèmes avec les fichiers extraits à l’aide de WinRAR.

    Si vous entendez un « bruit sec », potentiellement accompagné
    d’un rétroéclairage allumé pendant une fraction de seconde, il
    y a un problème matériel avec votre console (comme un câble de
    rétroéclairage déconnecté). Vous pourrez peut-être faire démarrer
    votre console en la tenant à certains angles.

    Ma console reste bloquée sur un écran noir avec le voyant
    d’alimentation bleu qui reste allumé

    Les étapes ci-dessous peuvent être tentées dans n’importe quel ordre, mais sont répertoriées de la moins longue à la plus longue.

    1. Éteignez votre console, retirez la carte SD, réinsérez-la, puis allumez votre console.
    2. Éteignez votre console, éjectez la cartouche de jeu si elle est insérée, allumez votre console, puis attendez jusqu’à dix minutes. Si votre console démarre dans les dix minutes, le problème a été résolu et il est peu probable qu’il se reproduise

    3. Renommez le dossier Nintendo 3DS de votre carte SD en Nintendo 3DS_BACKUP, puis essayez de démarrer. Si votre console démarre avec succès, il y a un problème dans votre dossier Nintendo 3DS. Essayez d’effacer HOME Menu extdata :

      • Accédez à /Nintendo 3DS///extdata/00000000/

      • Supprimez le dossier correspondant à votre région 3DS

        • EUR Région : 00000098
        • Région JPN : 00000082
        • Région États-Unis : 0000008f
        • Région CHN : 000000A1
        • Région KOR : 000000A9
        • Région TWN : 000000B1

    4. Essayez de démarrer en mode de récupération et de mettre à jour votre système :

      • Éteignez votre console
      • Maintenir (épaule gauche) + (épaule droite) + (D-pad vers le haut) + (A)
      • Allumez votre console
      • Si vous avez réussi, la console démarrera à partir d’un écran « Mettre à jour votre système »
    5. Suivez le guide CTRTransfer
    6. Pour plus d’aide, demandez de l’aide sur Nintendo Homebrew sur Discord

    Message d’erreur au démarrage

    « Une erreur s’est produite : Échec de l’application de 1 correctif FIRM » ou « Une exception s’est produite -- Processus en cours : pm »

    Votre version de Luma3DS est obsolète. Téléchargez la dernière version
    de Luma3DS et placez boot.firm à la racine de votre carte SD, en remplaçant
    tout fichier existant. Assurez-vous d’extraire le fichier ZIP avec un outil
    autre que WinRAR, car il est connu pour causer des problèmes avec les fichiers liés à 3DS.

    "Impossible de monter CTRNAND ou de charger le CTRNAND FIRM. Veuillez en utiliser un externe.

    Il y a un certain nombre de raisons pour lesquelles cela pourrait se produire.
    Dans tous les cas, cette erreur peut généralement être corrigée en suivant le guide CTRTransfer.

    « Une erreur s’est produite. Maintenez le bouton POWER enfoncé pour couper l’alimentation...

    Les gestionnaires d’exceptions ARM11 sont désactivés ou le micrologiciel personnalisé
    n’est pas installé. Essayez d’activer les gestionnaires d’exceptions ARM11 : + Éteindre
    votre console + Maintenir (Sélectionner) + Allumer votre console, tout en maintenant (Sélectionner)
    + Si la case « Désactiver les gestionnaires d’exceptions ARM11 » est cochée, décochez-la

    Il manque des applications installées dans le menu HOME

    Cela peut être dû à diverses raisons, mais très probablement parce que votre carte SD
    n’est pas lue par le système. Vous pouvez vérifier si votre SD est en cours de lecture
    en maintenant SELECT au démarrage et en vérifiant le texte jaune sur l’écran inférieur;
    s’il indique « Booted from CTRNAND via B9S », alors votre console démarre à partir de la
    mémoire interne et non à partir de la carte SD. Si tel est le cas, essayez les étapes
    ci-dessous, qui sont énumérées du plus facile au plus difficile :

    1. Éteignez votre console, retirez la carte SD, réinsérez-la, puis allumez votre console
    2. Éteignez votre console, retirez la carte SD, insérez-la dans votre ordinateur, téléchargez la dernière version de Luma3DS, extrayez boot.firm du Luma3DS.zip et placez-le à la racine de votre carte SD (en remplaçant tout fichier existant)
    3. Éteignez votre console, retirez la carte SD, insérez-la dans votre ordinateur et reformulez votre carte SD en fonction du système d’exploitation de votre ordinateur : Windows, macOS, Linux (cela effacera les données de votre carte SD)
    4. Testez votre carte SD pour détecter les erreurs en suivant le guide en fonction du système d’exploitation de votre ordinateur : Windows, Linux, macOS. Si votre carte SD est marquée comme défectueuse, vous devrez remplacer votre carte SD
    5. Il se peut que l’emplacement de votre carte SD soit cassé. Rejoignez Nintendo Homebrew sur Discord pour obtenir de l’aide supplémentaire

    Écran bleu « BOOTROM ERROR »

    Votre console est probablement en briques dures. Vous devrez acheter un flashcart
    ntrboot pour réinstaller boot9strap afin d’essayer de réparer votre console.
    Cela peut également indiquer un problème matériel qui ne peut pas être résolu.
    Dans tous les cas, rejoignez Nintendo Homebrew sur Discord pour obtenir de l’aide.
    + Il est également possible que quelqu’un ait défini un écran de démarrage qui
    ressemble à une brique. Essayez de laisser votre console allumée, en attendant
    sur l’écran bleu, pendant cinq minutes.

    Une autre erreur

    Veuillez prendre une photo de l’erreur et rejoindre Nintendo Homebrew sur Discord pour obtenir de l’aide.

    Problèmes logiciels sur les consoles avec micrologiciel personnalisé

    La fonctionnalité DSi / DS est cassée ou a été remplacée par Flipnote Studio
    1. Téléchargez la dernière version de TWLFix-CFW (le fichier .3dsx)
    2. Éteignez votre console
    3. Créez un dossier nommé 3ds à la racine de votre carte SD si elle n’existe pas déjà
    4. Copiez TWLFix-CFW.3dsx dans le dossier /3ds/ de votre carte SD
    5. Réinsérez votre carte SD dans votre console
    6. Ouvrez le lanceur Homebrew
    7. Lancez TWLFix-CFW à partir de la liste des homebrews
    8. Appuyez sur (A) pour désinstaller les titres TWL cassés
    9. Appuyez sur (Démarrer) pour redémarrer la console

    10. Mettez à jour votre console en allant dans les paramètres système, puis dans « Autres paramètres », puis en allant tout à droite et en utilisant « Mise à jour du système »

      • La mise à jour verra que les titres essentiels de TWL ont été désinstallés, et les téléchargera et les réinstallera
    11. Une fois la mise à jour terminée, appuyez sur « OK » pour redémarrer la console
    La fonctionnalité de la console virtuelle GBA et/ou du mode sans échec est défectueuse

    Votre console exécute Luma3DS 6.6 ou une version antérieure, probablement
    via arm9loaderhax. Vous devez suivre A9LH vers B9S pour mettre à jour votre
    console vers un environnement de firmware personnalisé moderne.

    Les jeux en mode mémoire étendue (Pokemon Sun/Moon, Smash, etc.) ne fonctionnent pas

    Cela peut se produire après un CTRTransfer ou un changement de région sur Old 3DS / 2DS. Suivez les instructions ici pour résoudre ce problème (en ignorant les étapes 3, 4, 5 et 6).

    Écran d’exception lors du démarrage/chargement d’une application

    Recherchez votre écran d’exception sur cette page. Si vous n’avez pas
    réussi à trouver votre erreur ou si les instructions n’ont pas fonctionné,
    rejoignez Nintendo Homebrew sur Discord pour obtenir de l’aide.

    Pour désinstaller menuhax67, téléchargez la dernière version de menuhax67
    (le .zip menuhax), puis suivez la section « Désinstaller menuhax67 » ici.

    Autres dépannages

    Effacer HOME Menu extdata
    1. Éteignez votre console
    2. Insérez votre carte SD dans votre ordinateur
    3. Accédez au dossier /Nintendo 3DS/"ID0"/"ID1"/extdata/00000000/ sur votre carte SD

    4. Supprimez le dossier correspondant à votre région 3DS :

      • EUR Région : 00000098
      • Région JPN : 00000082
      • Région États-Unis : 0000008f
      • Région CHN : 000000A1
      • Région KOR : 000000A9
      • Région TWN : 000000B1
    5. Réinsérez votre carte SD dans votre console
    Effacer les données du thème du menu HOME
    1. Éteignez votre console
    2. Insérez votre carte SD dans votre ordinateur
    3. Accédez au dossier /Nintendo 3DS/"ID0"/"ID1"/extdata/00000000/ sur votre carte SD

    4. Supprimez le dossier correspondant à votre région 3DS :

      • EUR Région : 000002ce
      • JPN Région : 000002cc
      • Région États-Unis : 000002cd
    5. Réinsérez votre carte SD dans votre console
    Saisie manuelle du Homebrew Launcher

    Si l’application Homebrew Launcher ne vous parvient pas dans
    votre menu HOME, vous pouvez suivre ces instructions pour entrer
    manuellement dans le Homebrew Launcher. (Vous aurez besoin de
    boot.3dsx et boot.firm à la racine de votre carte SD.)

    1. Lancez l’application Mode téléchargement
    2. Attendez de voir les boutons Nintendo 3DS et Nintendo DS
    3. Appuyez sur (épaule gauche) + (D-Pad bas) + (Sélectionner) en même temps pour ouvrir le menu Rosalina
    4. Sélectionnez « Options diverses »
    5. Sélectionnez "Changer le hb. titre de l’application actuelle.
    6. Appuyez sur (B) pour continuer
    7. Appuyez sur (B) pour revenir au menu principal de Rosalina
    8. Appuyez sur (B) pour quitter le menu Rosalina
    9. Appuyez sur (Accueil) pour suspendre Mode téléchargement
    10. Appuyez sur le bouton « Fermer » sur l’écran inférieur pour fermer Mode téléchargement
    11. Relancez l’application Mode téléchargement

    12. Votre console doit charger le Homebrew Launcher

      • Si votre console est bloquée sur l’écran de démarrage de chargement, il vous manque boot.3dsx à partir de la racine de votre carte SD
    Désactiver le contrôle parental

    Vous pouvez désactiver la fonction de contrôle parental en allant
    dans Paramètres système -> Contrôle parental et en insérant le code PIN,
    puis en appuyant sur « Effacer les paramètres », puis sur « Supprimer » pour
    le supprimer. Cependant, si vous ne connaissez pas le code PIN et que vous ne
    pouvez donc pas accéder aux paramètres de la console, vous devrez le désactiver.
    Pour ce faire, vous devez obtenir la clé principale de votre console (mkey) :

    1. Aller sur ce site

    2. Remplissez les cases suivantes avec les informations :

      • Type d’appareil : Sélectionnez « 3DS » (il en va de même si vous utilisez une 2DS, une nouvelle 3DS (XL/LL) ou une nouvelle 2DS (XL/LL))
      • Date système : le jour et le mois où l’horloge de votre console est réglée sur
      • Numéro de demande : Peut être obtenu en appuyant sur « Code PIN oublié » puis « J’ai oublié » dans l’écran de contrôle parental
    3. Une fois que vous avez obtenu votre mkey, appuyez sur OK sur l’écran où vous avez obtenu votre numéro de demande, puis saisissez la clé maîtresse
    4. Appuyez sur « Effacer les paramètres », puis sur « Supprimer » pour supprimer toutes les données du contrôle parental
